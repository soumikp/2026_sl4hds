Training and Testing Splits

To evaluate our model’s performance on unseen data, we must split our dataset into a training set (used to build the model) and a testing set (used to evaluate it). We’ll use an 80/20 split, meaning 80% of the data will be used for training and 20% for testing.

We use the createDataPartition function from the caret package to create a stratified split, which ensures that the proportion of each species is the same in both the training and testing sets.

Note: For this analysis, we are only using Sepal.Length and Sepal.Width as our predictors (train_x, test_x).

# Set a seed for reproducibility
set.seed(42)

# Create indices for the 80% training data
train_indices <- createDataPartition(iris$Species, p = 0.8, list = FALSE)

# Create the training feature set (X) and target (y)
train_x <- as.matrix(iris |> dplyr::select(-Species))[train_indices, c("Sepal.Length", "Sepal.Width")]
train_y <- as.matrix(iris |> dplyr::select(Species))[train_indices, ]

# Create the testing feature set (X) and target (y)
test_x <- as.matrix(iris |> dplyr::select(-Species))[-train_indices, c("Sepal.Length", "Sepal.Width")]
test_y <- (iris |> dplyr::select(Species))[-train_indices, ]

Fitting SVMs

Now we’ll train three SVM models with different kernels. We use the train function from caret, which helps automate hyperparameter tuning using cross-validation. We’ll use 10-fold repeated cross-validation to get a robust estimate of model performance. We also pre-process the data by centering and scaling it, which is standard practice for SVMs.

# Define the training control method
train_control <- trainControl(method="repeatedcv", number=10)

## Train a Linear SVM
# We tune the cost parameter 'C'
svm_linear <- train(x = train_x, y = train_y,
                    method = "svmLinear",
                    trControl = train_control,
                    preProcess = c("center","scale"),
                    tuneGrid = expand.grid(C = c(1/16, 1/8, 1/4, 1/2, 1, 2, 4))) 

## Train a Polynomial SVM
# We tune 'degree', 'scale', and 'C'
svm_poly <- train(x = train_x, y = train_y,
                  method = "svmPoly",
                  trControl = train_control,
                  preProcess = c("center","scale"),
                  tuneGrid = expand.grid(degree = c(1:4),
                                         scale = c(0.001, 0.010, 0.1, 1),
                                         C = c(1/16, 1/8, 1/4, 1/2, 1, 2, 4))) 

## Train a Radial Basis Function (RBF) SVM
# We tune 'sigma' and 'C'
svm_radial <- train(x = train_x, y = train_y,
                    method = "svmRadial",
                    trControl = train_control,
                    preProcess = c("center","scale"),
                    tuneGrid = expand.grid(sigma = c(0.001, 0.010, 0.1, 1),
                                           C = c(1/16, 1/8, 1/4, 1/2, 1, 2, 4)))

Conclusion

The SVM with the Radial Basis Function (RBF) kernel was the top-performing model. This suggests that the decision boundary between the species based on sepal data is non-linear, and the RBF kernel was most effective at capturing this complex relationship. This exercise demonstrates the importance of testing different kernels and tuning hyperparameters to find the best model for a given dataset.
